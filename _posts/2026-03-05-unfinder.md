# UN (Unity-Native) Fast Object Finder SDK

![](https://img.shields.io/badge/unity-2021.3%2B-black)
![](https://img.shields.io/badge/license-MIT-blue)

> **UNFinder**는 Unity에서 **`GameObject.Find`** 와 **`GetComponent`** 를 대체하기 위한  
> **고성능 오브젝트 조회 프레임워크**.  
> **O(1)** 상수 시간으로 객체를 탐색할 수 있도록 설계되었음.

**문자열 해싱**, **순수 C# 메모리 딕셔너리**,
그리고 **`ConditionalWeakTable`** 을 결합하여  
UNFinder는 **GC 할당 없이 객체를 조회**하면서
Unity의 **네이티브 C++ 마샬링 오버헤드**를 우회함.

빌드 과정에서 SDK는 자동으로 씬에 숨겨진 트래커를 삽입.
개발자는 기존과 동일한 방식으로   
객체를 조회할 수 있으며 내부적으로는 **완전히 캐싱된 런타임 레지스트리**를 활용.

# Core Features

### O(1) 고속 조회

`GameObject.Find` 내부에서 발생하는
**네이티브 C++ 마샬링 비용**을 제거하고

객체를 **순수 C# 메모리 구조**에서 직접 조회함.

---

### Zero GC Allocation

캐시 히트 시 **가비지 메모리가 전혀 생성되지 않으며**,  
Unity의 **Fake-Null 브리지 오버헤드** 또한 발생하지 않음.

---

### Memory Leak 방지

컴포넌트 캐싱은 **`ConditionalWeakTable`** 을 사용.

따라서 연결된 `GameObject` 가 파괴될 경우
캐시된 항목 역시 **자동으로 해제.**

---

### Auto Baking System

빌드 단계에서 SDK가 씬을 분석하고

**모든 오브젝트를 자동으로 런타임 캐시 레지스트리에 등록.**

개발자가 별도의 수동 설정을 할 필요가 없음.

---

# API Reference & Usage

API는 **Unity에서 자주 사용하는 객체 조회 패턴을 그대로 대체**할 수 있도록 설계되었음.

---

## Object & Component Lookup

객체가 캐시에 존재할 경우 **O(1)** 시간으로 반환.

캐시에 존재하지 않을 경우
**Unity의 기본 조회를 lazy fallback으로 실행한 뒤 캐싱.**

```csharp
// GameObject.Find("Player") 대체
GameObject player = UN.Find("Player");

// 컴포넌트 조회 + 캐싱
PlayerController controller = UN.FindComponent<PlayerController>("Player");
```

---

## Dynamic Instantiation

생성된 Prefab은 **즉시 조회 캐시에 등록.**

또한 Unity가 기본적으로 붙이는
`"(Clone)"` 접미사를 제거하여

원본 오브젝트 이름을 그대로 유지함.

```csharp
GameObject enemyInstance =
    UN.Instantiate(enemyPrefab, spawnPos, Quaternion.identity);

EnemyAI ai =
    UN.Instantiate<EnemyAI>(enemyAIPrefab, spawnPos, Quaternion.identity);
```

---

## Rename & Manual Binding

런타임에서 오브젝트 이름을 변경할 경우
**캐시 무결성을 유지하기 위해 제공된 API를 사용해야 함.**

```csharp
// 수동 등록
UN.Bind(dynamicObject, "NewDynamicBoss");

// 캐시 해시를 동기화하며 이름 변경
UN.Rename(playerObject, "Player_Awakened");
```

# Lifecycle & Callback Mechanism

**UNFinder의 가장 큰 장점 중 하나는
개발자가 객체 등록 / 해제를 직접 관리할 필요가 없다는 점임.**

SDK는 Unity의
**빌드 단계 콜백**과 **런타임 라이프사이클 훅**을 활용하여
자동으로 동기화.

```
Build Phase
-----------
IProcessSceneWithReport.OnProcessScene
→ 씬의 모든 Transform 순회
→ 숨겨진 UNTracker 컴포넌트 주입
→ ~UN_AutoRegistry 바인딩 생성

Bootstrapping Phase
-------------------
DefaultExecutionOrder(-32000)
→ UNSceneRegistry.Awake()
→ FNV-1a 해시로 씬 객체 해싱
→ _fastBucket 에 등록

Runtime Lookup
--------------
UN.Find("Player")
→ 즉시 Dictionary 반환 (Native 호출 없음)

Destroy Phase
-------------
UNTracker.OnDestroy()
→ UN.RemoveFromCache()
→ 캐시 자동 정리
```

# Performance Comparison

씬 규모가 커질수록
`GameObject.Find` 는 **O(N) 계층 탐색** 때문에
성능이 선형적으로 감소함.

UNFinder는 씬 크기와 관계없이
**항상 O(1) 상수 시간 조회를 유지할 수 있도록 설계되었음.**

<img width="1600" height="960" alt="PerformGraph" src="https://github.com/user-attachments/assets/4a7c5e16-473a-4242-a73c-625392d421a9" />

### Default Unity API

* 네이티브 **C++ 마샬링**
* 전체 Hierarchy 탐색
* 반복 호출 시 성능 저하

### UNFinder SDK

* 순수 **C# Dictionary 조회**
* **0 byte GC Allocation**
* 결정적 **O(1)** 객체 탐색
